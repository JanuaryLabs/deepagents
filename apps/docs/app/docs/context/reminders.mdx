title: Reminders
description: Inject situational context into model-visible carriers so the LLM stays aware of hidden runtime facts, active skills, and recent conversation signals

Reminders inject hidden text into the model conversation to give the LLM situational awareness — current date, active skills, recent tool behavior, recurring instructions. Depending on the target, the text can merge into a user message, wrap a tool result, or splice a synthetic mid-loop steer message into a streaming turn. In every case the text is invisible to the end user but visible to the model.

## How Reminders Work

Every reminder attaches hidden text to one model-visible carrier. Three decisions control when and how:

1. **When** — immediate (at message creation) or conditional (at `engine.save()` / stream evaluation, gated by a predicate)
2. **Where** — `target: 'user'`, `target: 'tool-output'`, or `target: 'steer'`
3. **How** — inline (appended as a `<system-reminder>` tag inside the last text part) or as a separate text part

## Immediate Reminders

Pass reminders directly to `user()`. They resolve at message creation time and receive the message content.

```ts
import { user, reminder } from '@deepagents/context';

// Static text
user('Hello', reminder('Keep responses under 100 words'));

// Factory — receives message content
user(
  'Explain polymorphism',
  reminder((ctx) => `User said: "${ctx.content}". Tailor to their level.`),
);
```

A factory that returns an empty string skips injection entirely — useful for conditional logic without the full predicate system.

```ts
user(
  'deploy to prod',
  reminder((ctx) =>
    ctx.content.includes('prod') ? 'Ask for confirmation before destructive actions' : ''
  ),
);
```

### Inline vs Part Mode

Reminders render in one of two modes:

- **Inline** — appended as `<system-reminder>` tags to the last text part.
- **Part** — added as a separate text part on the user message.

The default depends on the input type:

| Input | Default `asPart` |
| --- | --- |
| `reminder(text: string \| factory)` | `false` (inline) |
| `reminder(fragment: ContextFragment)` | `false` (inline) |

Reminders default inline so the model sees the context next to the user's words. Set `asPart` to true when structured content needs to stay in a standalone text part:

```ts
const partMode = true;

reminder('hint'); // inline
reminder(workflow({ task: 'Deploy', steps: ['…'] })); // inline
reminder(workflow({ task: 'Deploy', steps: ['…'] }), { asPart: partMode }); // forced part
```

Use part mode for structured data (dates, skill lists, fragments) to keep reminder text separate from the user's message.

## Conditional Reminders

Add a `when` predicate to create reminders that fire based on turn count, message content, assistant/tool history, token usage, or time. Pass them to `engine.set()`; the engine evaluates them at `save()` time.

```ts
import { reminder, everyNTurns, user } from '@deepagents/context';

engine.set(
  reminder('Remember to cite sources', { when: everyNTurns(3) }),
  user('Tell me about quantum computing'),
);
```

For the default `user` target, the engine applies conditional reminders to the **last pending user message** during `save()`. It builds a `WhenContext` with the current turn count, message content, usage totals, message counts, and recent assistant history, then evaluates each predicate.

```ts
import { reminder, contentIncludes, afterTurn, and, user } from '@deepagents/context';

// Only fires when user mentions code AND after the first 2 turns
engine.set(
  reminder('Include code examples in your response', {
    when: and(contentIncludes(['code', 'example', 'snippet']), afterTurn(2)),
  }),
  user('Show me how to use async/await'),
);
```

See [Predicates](/docs/context/predicates) for the full predicate catalog and composition rules.

### Reminder Targets

Conditional reminders can fire in three places:

| Target | When it runs | What the model sees |
| --- | --- | --- |
| `user` | During `engine.save()` on the last pending user message | Reminder text merged into that user message |
| `tool-output` | Right after a tool `execute()` resolves | Wrapped tool output on the next model step, with the same wrapped value persisted in the chain |
| `steer` | Mid-loop between streaming steps, after the model has already emitted content | A synthetic hidden user message containing one or more `<system-reminder>` parts |

`user` is the default target. `tool-output` and `steer` require a `when` predicate.

```ts
import {
  afterTurn,
  everyNTurns,
  reminder,
  streamStepsExceed,
} from '@deepagents/context';

engine.set(
  reminder('Double-check filesystem side effects', {
    when: afterTurn(0),
    target: 'tool-output',
  }),
  reminder('Checkpoint before taking another tool step', {
    when: streamStepsExceed(2),
    target: 'steer',
  }),
  reminder('Cite sources every third turn', {
    when: everyNTurns(3),
  }),
);
```

Use `tool-output` when the reminder should travel with a tool result. Use `steer` when the agent needs a hidden nudge during a multi-step streamed turn without waiting for the next user message. Steer reminders only fire after the model has produced at least one streamed step with content. When a steer reminder fires, the engine persists the same assistant/steer/assistant split that the model saw, and constant-true predicates fire once per stream instead of repeating on every later step.

## ReminderContext

Both immediate and conditional reminder factories receive a context object:

```ts
interface ReminderContext {
  content: string;            // Plain text of the user message
  turn?: number;              // Current user turn count
  lastMessageAt?: number;     // Timestamp of last persisted user message
  lastMessage?: UIMessage;    // Last persisted user message (with metadata)
  currentMessage?: UIMessage; // Pending user message for the current turn
  chat?: StoredChatData;      // Active chat record
  usage?: LanguageModelUsage; // Accumulated usage metadata
  branch?: string;            // Active branch name
  elapsed?: number;           // Milliseconds since last persisted user message
  messageCount?: number;      // Total message count across all roles
  lastAssistantMessage?: UIMessage; // Last persisted assistant message
  streamStepNumber?: number;      // Current streamed step number during steer evaluation
  streamToolCallCount?: number;   // Completed tool calls seen in the current streamed turn
  streamTotalTokens?: number;     // Running token total for the current streamed turn
}
```

Conditional reminders populate these fields at `save()` time. `target: 'steer'` reminders additionally receive the stream-scoped fields while a turn is in flight. Immediate reminders receive only `content`.

## Stripping Reminders

`stripReminders()` removes all reminder text and metadata from a message. Use it to display clean message history or export conversations.

```ts
import { stripReminders } from '@deepagents/context';

const clean = stripReminders(message);
// No <system-reminder> tags, no reminder metadata
```
