This guide walks you through installing Text2SQL and running your first natural language to SQL query.

## Installation

```bash
npm install @deepagents/text2sql
```

Database-specific drivers are peer dependencies. Install the one that matches your database:

```bash
npm install pg        # PostgreSQL
npm install mssql     # SQL Server
```

SQLite uses the built-in `node:sqlite` module and requires no additional driver.

## Basic Setup

Text2SQL requires six components:

1. **Adapter** - Connects to your database (PostgreSQL, SQLite, SQL Server, MySQL/MariaDB, or BigQuery)
2. **Grounding** - Configures database introspection (see [Grounding](/docs/text2sql/grounding) for advanced options)
3. **Model** - Any AI SDK model compatible with `@deepagents/agent`
4. **Filesystem** - A `just-bash` filesystem used by result tools
5. **Context Factory** - Creates a `ContextEngine` with your chosen store, `chatId`, and `userId`
6. **Version** - A string used to manage cache invalidation (bump when schema changes)

Start by creating the shared model, filesystem, and context factory:

```typescript
import { groq } from '@ai-sdk/groq';
import { InMemoryFs } from 'just-bash';

import { ContextEngine, InMemoryContextStore } from '@deepagents/context';

const store = new InMemoryContextStore();
const model = groq('openai/gpt-oss-20b');
const filesystem = new InMemoryFs();

const context = (...fragments) => {
  const engine = new ContextEngine({
    store,
    chatId: 'chat-123',
    userId: 'user-456',
  });
  engine.set(...fragments);
  return engine;
};
```

Reuse `model`, `filesystem`, and `context` in the adapter-specific examples below.

```typescript
import { BigQuery as BigQueryClient } from '@google-cloud/bigquery';

import { Text2Sql } from '@deepagents/text2sql';
import { BigQuery } from '@deepagents/text2sql/bigquery';
import * as bigquery from '@deepagents/text2sql/bigquery';

const projectId = process.env.BQ_PROJECT_ID!;

  filesystem,

    datasets: ['analytics'],
    execute: async (sql) => {
      const [job] = await client.createQueryJob({ query: sql });
      const [rows] = await job.getQueryResults();
      return rows;
    },
    validate: async (sql) => {
      await client.createQueryJob({
        query: sql,
        dryRun: true,
        useQueryCache: false,
      });
    },
    grounding: [bigquery.tables(), bigquery.info()],
  }),
});
```

## Your First Query

### Generate SQL Only

Use `toSql()` to generate SQL without executing it:

```typescript
const sql = await text2sql.toSql(
  'Show me the top 10 customers by total purchases',
);

console.log(sql);
// SELECT c.CustomerId, c.FirstName, c.LastName, SUM(i.Total) as TotalPurchases
// FROM Customer c
// JOIN Invoice i ON c.CustomerId = i.CustomerId
// GROUP BY c.CustomerId
// ORDER BY TotalPurchases DESC
// LIMIT 10
```

### Chat with Streaming

Use `chat()` for streaming responses. Conversation identity comes from the `context` factory you created above:

```typescript
const stream = await text2sql.chat([
  { role: 'user', content: 'What are the most popular genres?' },
]);

for await (const chunk of stream) {
  if (chunk.type === 'text-delta') {
    process.stdout.write(chunk.textDelta);
  }
}
```

## Understanding the Response

When Text2SQL processes a query, it:

1. **Runs grounding** - Introspects your database schema (tables, columns, relationships, indexes, constraints).
2. **Builds SQL context** - `toSql()` uses cached schema fragments, while `chat()` also resolves the stored conversation thread from your context store.
3. **Generates SQL** - The model maps the request to the available schema.
4. **Validates** - The adapter validates the generated SQL before it is returned or executed.
5. **Returns results** - `toSql()` returns the SQL string; `chat()` streams an end-user answer.

`chat()` also injects Text2SQL's built-in SQL workflow guidance and any extra fragments you add in your `context` factory.

## Environment Variables

Text2SQL uses Groq for AI inference. Set your API key:

```bash
export GROQ_API_KEY=your-api-key
```
